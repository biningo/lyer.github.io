Vagrant总结

lyer 发布于 2021-09-13 收录于 Vagrant
什么是Vagrant Vagrant是一个配置虚拟机的工具，底层还是需要依赖虚拟机比如VirtualBox来创建虚拟机的，Vagrant只是调用了虚拟机提供的接口来快速配置虚拟机 Vagrant主要用于配置开发环境，而Docker用于在生产环境上部署 ​ Vagrant基本使用 首先我们需要创建一个工作目录 mkdir test 然后需要初始化一个 Vagrantfile ，后面的hashicorp/bionic64就代表想要启动一个什么样的box，box其实就和docker镜像一样，是一种对虚拟机以及配置的打包 vagrant init hashicorp/bionic64 启动虚拟机 vagrant up 销毁删除虚拟机 vagrant destroy 暂停/恢复虚拟机，实际使用此命令来暂停和启动虚拟机 vagrant suspends vagrant resume 关机/启动虚拟机，如果虚拟机没创建则会自动创建 vagrant halt vagrant up 修改Vagrantfile之后使之生效，相当于halt+up重启机器 vagrant reload 进入虚拟机 vagrant ssh 查看当前目录下虚拟机的状态，查看全局虚拟机的状态 vagrant status vagrant global-status ​ 如何导入box 对于创建box对应的虚拟机，我们需要下载box然后才能基于box进行创建虚拟机 我们可以在Vagrantfile文件里面进设置相关的配置 config.vm.box = "ubuntu/trusty64" #回去官方的box仓库下载对应名字的box config.vm.box = "http://example.com/virtualbox.box" #指定url config.vm.box = "~/Downloads/virtualbox.box" #指定本地box磁盘路径 我们在初始化Vagrantfile的时候也可以指定box vagrant init ubuntu/trusty64 vagrant init http://example.com/virtualbox.box vagrant init ~/Downloads/virtualbox.box ​ ​

微服务部署方式

lyer 发布于 2021-09-13 收录于 分布式和微服务
蓝绿部署 蓝绿部署的过程如下: 将集群分为两部分: A和B，在没有升级版本时两个集群都是一致对外提供服务 此时如果系统升级，则将网关处删除A组集群，让B组集群单独提供服务 然后升级A集群到最新版本，再将网关指向A同时删除集群B，此时有A单独提供服务 然后升级集群B，升级完成之后再将集群B添加到网关，此时两部分集群就都可以对外提供服务了 蓝绿部署的缺点是: 服务升级过程中集群资源会减少一半，以此此部署方式不适合在高流量的时候发布 服务是全量升级，如果新版本服务有问题则会影响所有用户 ​ 红黑部署 红黑部署和蓝绿部署的区别是红黑部署在版本升级的时候会重新创建一个相同大小的新版本集群，然后再将网关流量指向新版本集群同时删除老集群 可以看到红黑部署解决了蓝绿部署升级时集群资源减少为一半的问题，但是缺点是升级的时候需要两倍的集群资源，同时也属于全量部署 ​ 灰度发布/金丝雀发布 在发布过程中一部分用户继续使用老版本，一部分用户使用新版本，不断地扩大新版本的访问流量，终实现老版本到新版本的过度 ​ 巨人肩膀 蓝绿红黑灰|常用的发布方式

为什么Linux空目录是4k大小

lyer 发布于 2021-09-01 收录于 十万个为什么
由这个问题还可以衍生出一个问题 为什么文件(一切皆文件，包括目录也是一个文件)大小的单位都是4k的倍数? 要解决这个文件首先我们需要搞清楚Linux的VFS虚拟文件系统和inode机制，以及Linux操作系统是如何管理磁盘的如何从磁盘读写数据的 Linux为了提高读写磁盘的效率，将磁盘分割为一个个4k的块，然后以块为单位读写。操作系统会在inode中设置文件数据保存的块的索引，读取的时候再以块为单位读取到内存 所以一个文件大小的单位都可以被4k整除，因为操作系统会按磁盘块来进行申请空间 为什么空的普通文件大小为0k而空目录文件大小为4k ？ 因为一个新创建的目录下会保存.和..这两个目录 一个是到当前目录的目录项 一个是到上一层目录的项 所以任何一个新的目录初始的大小一定是4k开始

crontab命令执行定时任务

lyer 发布于 2021-08-29 收录于 Linux
crontab概述 crontab用来在Linux下执行定时任务的工具，Linux下定时任务主要分为系统任务和用户自定义任务两大类 系统执行的工作：系统周期性所要执行的工作，如备份系统数据、清理缓存 个人执行的工作：某个用户定期要做的工作，例如每隔10分钟检查邮件服务器是否有新信，这些工作可由每个用户自行设置 ​ crontab基本用法 使用之前我们需要设置一个crontab配置文件的默认编辑器 export EDITOR="vim" 然后我们可以使用如下命令编辑crontab配置文件，设置定时任务 crontab -e #打开当前用户的配置文件进行编辑 sudo crontab -u test -e #指定用户 配置文件里的写法就是先写cron时间表达式，然后再追加需要执行的任务 * * * * * echo "hello,world" >> /temp/log 我们查看配置文件的定时任务可以使用如下命令查看 crontab -l 可以使用如下命令删除配置文件所有的定时任务 crontab -r ​ cron时间表达式 * * * * * | | | | | | | | | +----- 星期中星期几 (0 - 6) (星期天 为0) | | | +---------- 月份 (1 - 12) | | +--------------- 一个月中的第几天 (1 - 31) | +-------------------- 小时 (0 - 23) +------------------------- 分钟 (0 - 59) * 代表整个时间段
